# Data Model: Nix Container Migration

**Feature**: 001-nix-container-migration
**Date**: 2026-03-18

---

## Entities

### 1. Nix Flake (`nix/flake.nix`)

The top-level entry point. Declares inputs (nixpkgs, nix2container) and outputs (all images).

**Fields**:

- `inputs.nixpkgs`: Pinned nixpkgs revision (URL + hash in `flake.lock`)
- `inputs.nix2container`: Pinned nix2container revision
- `outputs.packages.{system}.core`: Core image derivation
- `outputs.packages.{system}.dev`: Development image derivation
- `outputs.packages.{system}.ecosystems.{name}`: Per-ecosystem image derivations

**Relationships**: All ecosystem images depend on `core`. `dev` depends on any ecosystem image.

---

### 2. Core Image Definition (`nix/core.nix`)

Replaces `Dockerfile.updater-core`. Produces the base OCI image.

**Fields / layers**:

- `systemLayer`: git, git-lfs, bzr, hg, gnupg2, openssh, ca-certificates, gcc, make, gmp, zlib, unzip, zstd, file, libyaml, locales
- `rubyLayer`: Ruby 3.4.x, RubyGems, Bundler
- `gitShimLayer`: git-shim binary at `~/bin/git`
- `userSetup`: `/etc/passwd`, `/etc/group`, `/etc/shadow` with `dependabot:1000:1000`
- `gemLayer`: Bundled updater gems from `updater/Gemfile.lock`
- `sourceLayer`: `common/`, `omnibus/`, ecosystem stubs, `updater/`

**Environment variables**:

- `DEPENDABOT_HOME=/home/dependabot`
- `DEPENDABOT=true`
- `GIT_LFS_SKIP_SMUDGE=1`
- `DEPENDABOT_NATIVE_HELPERS_PATH=/opt`
- `PATH=$DEPENDABOT_HOME/bin:$PATH` (including Nix store paths)
- `LC_ALL=en_US.UTF-8`, `LANG=en_US.UTF-8`

**Config**:

- `User`: `dependabot`
- `WorkingDir`: `/home/dependabot/dependabot-updater`
- `Cmd`: `["bin/run"]`

**Validation rules**: Must contain all binaries listed in FR-002. Ruby version must match FR-003.

---

### 3. Ecosystem Image Definition (`nix/ecosystems/{name}.nix`)

One per ecosystem. Layers tooling on top of core.

**Fields**:

- `fromImage`: Reference to core image
- `toolchainLayer`: Language runtime(s) and package manager(s) specific to this ecosystem
- `helperLayer`: Built native helpers from `{ecosystem}/helpers/build`
- `sourceLayer`: Ecosystem gem source (`{ecosystem}/lib/`, `common/`, `updater/`)
- `extraEnvVars`: Ecosystem-specific environment variables

**Categories by complexity**:

| Category | Ecosystems | Toolchain |
|----------|-----------|-----------|
| Minimal (core only) | silent, docker_compose, devcontainers, git_submodules, github_actions, dotnet_sdk, conda, elm, bazel | None or minimal binaries |
| Single runtime | go_modules (Go), cargo (Rust), swift (Swift), julia (Julia), pub (Dart/Flutter) | One language runtime |
| Runtime + helpers | bundler, npm_and_yarn, composer, hex, maven, gradle, nuget, terraform, opentofu, pre_commit, helm, vcpkg | Runtime + native helper build |
| Multi-runtime | python, uv (6 Python versions), bun (Node.js), npm_and_yarn (Node + npm + pnpm + yarn) | Multiple versions or tools |
| Binary-only | docker (cosign + regctl) | Third-party binaries |

**State transitions**: N/A (images are immutable build artifacts)

---

### 4. Development Image Definition (`nix/development.nix`)

Overlays debug/dev tools onto any ecosystem image.

**Fields**:

- `fromImage`: Any ecosystem image
- `devToolsLayer`: vim, strace, ltrace, gdb, shellcheck, libgit2, cmake, pkg-config
- `devConfig`: Bash PS1 prompt, .vimrc, unbundled gems (development group)

---

### 5. Dependency Pin Set (`nix/flake.lock`)

Auto-generated by `nix flake lock`. Pins all Nix inputs.

**Fields**:

- `nixpkgs.rev`: Git commit hash of pinned nixpkgs
- `nixpkgs.narHash`: Content hash for verification
- `nix2container.rev`: Git commit hash of pinned nix2container
- `nix2container.narHash`: Content hash

**Validation rules**: Must be updated via `nix flake update` (explicit action). Changes to `flake.lock` must be committed and reviewed like any other code change.

---

### 6. Shared Helper Library (`nix/lib/`)

Reusable Nix functions for building images.

**Functions**:

- `mkEcosystemImage { name, fromImage, toolchainPackages, helperBuildScript, envVars, sourceGlob }`: Builds an ecosystem image with standard layers (toolchain, helpers, source, env).
- `mkDevImage { fromImage, extraPackages }`: Overlays dev tools onto an ecosystem image.
- `mkUser { name, uid, gid, home }`: Creates passwd/group/shadow entries.
- `fetchGitShim { version, arch }`: Downloads and extracts the git-shim binary.

---

## Relationships

```
flake.nix
  ├── flake.lock (pins nixpkgs + nix2container)
  ├── core.nix → Core Image
  │     ├── lib/mkUser → user setup
  │     └── packages/git-shim.nix → git-shim binary
  ├── ecosystems/{name}.nix → Ecosystem Image
  │     ├── fromImage: Core Image
  │     ├── lib/mkEcosystemImage → shared builder
  │     └── {ecosystem}/helpers/build → native helper build
  └── development.nix → Dev Image
        ├── fromImage: any Ecosystem Image
        └── lib/mkDevImage → shared builder
```

## Version Management

Tool versions are managed in one of two places:

1. **nixpkgs pin** (`flake.lock`): Controls the version of every package from nixpkgs. Bumping nixpkgs bumps all packages atomically.
2. **Explicit version overrides** (in ecosystem `.nix` files): For tools that need a specific version different from what nixpkgs provides (e.g., `go_1_26` instead of default `go`).

A version bump workflow:

- To bump Go: Change `pkgs.go_1_26` to `pkgs.go_1_27` in `nix/ecosystems/go_modules.nix`.
- To bump all system tools: Run `nix flake update nixpkgs` and commit the new `flake.lock`.
- To bump Ruby: Change `pkgs.ruby_3_4` to `pkgs.ruby_3_5` in `nix/core.nix` (if major) or update nixpkgs pin (if patch).
